ronter를 이용하여 uri를 통한 페이지 이동 처리를 위해 사용(transfiler 후 a tag)

  방법
    router-view : router mapping component 내용
    router-link : uri address
    (App.vue file)
      ex)
      const router = new VueRouter({
        routes : [
          { path:'/', component: Home }
        ]
      })


  Dynamic Route
    일정한 패턴의 경로를 하나의 컴포넌트에 연결하는 방법, uri 경로 일부에 실행에 필요한 파라미터 값 포함된 경 유용
    (contacts.vue, ContactByNo.vue file)
      ex)
      연락처 상세보기 페이지   :no로 해당 위치에 저장된 문자열 파라미터로 값 획득
      { path:'/contacts/:no', component: ContactByNo },


  Nested Routes (중첩)
    하나의 컴포넌트가 다시 하위 컴포넌트를 포함하는 경우, watch속성을 이용하여 값이 변할때마다 재호출
    (contacts2.vue)
      ex )
      { path:'/contacts2', component: Contacts2,
        children:[
          {path:':no', component:ContactByNo}
        ]
      }

  Named Routes
    route 정보에 고유한 이름 부여 : uri경로가 아닌 부여된 route name으로 navigation
      ex)
      { path:'/home', component: Home, name:'home' }
      <router-link :to="{name:'home'}">Home </router-link>

  Router control -push method
    페이지 이동하기 전 이벤트 처리를 위해 사용
    push(location [, completeCallback] [, abortCallback])
      location         : 이동하고자하는 경로 (필수)
      completeCallback : navigation 완료 되었을때 호출 (필수x)
      abortCallback    : navigation 취소 되었을때 호출 (필수x)

      ex )
      this.$router.push('/home')
      this.$router.push({path:'/about'})
      this.$router.push({name:'contacts', params:{no:this.no}})
      this.$router.push(path:'/contacts', query:{pageno:1, pagesize:5})
